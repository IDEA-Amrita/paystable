## Product Boundary

Paystable is intentionally narrower than a payment orchestrator.

| Paystable does | Paystable does not |
|---|---|
| Stabilize webhook and verification signals. | Route payments across processors. |
| Detect amount mismatch and unresolved states. | Vault cards or handle PCI card data. |
| Deliver signed, retryable merchant callbacks. | Replace Razorpay, PayU, Cashfree, or Hyperswitch. |
| Keep a ledger for support and gateway disputes. | Automatically refund customers. |

## Core States

| State | Meaning |
|---|---|
| `PENDING` | Hold exists, no terminal evidence yet. |
| `VERIFYING` | Gateway signal received or status checks are active. |
| `CONFIRMED` | Stable success and amount match. |
| `FAILED` | Stable failure or TTL final check verified failure. |
| `MISMATCH` | Gateway success amount differs from the hold. |
| `INDETERMINATE` | Paystable could not reach a safe answer automatically. |
